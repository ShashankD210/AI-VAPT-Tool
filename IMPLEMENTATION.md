### Key Design Decisions

| Decision | Rationale |
|---|---|
| **Single-file Python entry** (`vapt_tool.py`) | Zero-import-path issues; run anywhere with `python3 vapt_tool.py` or `./vapt_tool.py` |
| **Scanner-plugin architecture** | Each scanner is a class with a `.run(target)` interface — easy to extend without touching core |
| **Lazy imports for heavy deps** | `fastapi`, `uvicorn`, `celery` imported at call-time so `--help` and `--version` work even without full env |
| **Graceful degradation** | Missing optional deps (Celery, PyYAML) produce a warning, not a crash |
| **`set -euo pipefail` installer** | Fail-fast shell script; trap for cleanup on error |

## 3. Installation & Setup

### 3.1 Automated Installer (Recommended)

```bash
cd "/home/johnwick/Projects/AI VAPT Tool"
sudo ./install.sh
```

The installer performs these steps automatically:

| Step | Action |
|---|---|
| 1 | Validates OS, RAM (≥4 GB), disk (≥10 GB), internet |
| 2 | Detects package manager (apt / yum / dnf) |
| 3 | Installs system deps: Python 3.9+, pip, venv, build tools, nmap, masscan, PostgreSQL, Redis, Nginx, Node.js |
| 4 | Creates Python virtualenv at `/opt/vapt-tool/venv` |
| 5 | Installs all Python packages from `/tmp/requirements.txt` |
| 6 | Creates full directory tree (`/opt/vapt-tool/{src,config,data,models,reports,logs,...}`) |
| 7 | Generates `config/config.yaml` + `.env` with random secrets |
| 8 | Configures PostgreSQL (user, DB, perf tuning) |
| 9 | Configures Redis (auth, memory limits) |
| 10 | Creates 4 systemd services: `vapt-tool`, `vapt-api`, `vapt-worker`, `vapt-beat` |
| 11 | Configures Nginx reverse-proxy |
| 12 | Creates backup script + daily cron job |
| 13 | Creates uninstall script |
| 14 | Creates `/usr/local/bin/vapt` CLI wrapper |
| 15 | (Optional) Trains initial AI models |
| 16 | Configures UFW or firewalld |
| 17 | Starts all services |
| 18 | Prints completion info with access URLs |

#### Installer Safety Features

- `set -euo pipefail` — exits on any error, undefined variable, or pipe failure
- `trap cleanup_on_error ERR` — rollback prompt on failure
- Passwords generated via `openssl rand -base64` — no hardcoded credentials
- Config files chmod 600 — owner-only read/write

### 3.2 Manual Python Setup (Development)

```bash
# Clone / extract project
cd "/home/johnwick/Projects/AI VAPT Tool"

# Create virtualenv
python3 -m venv venv
source venv/bin/activate

# Install core
pip install -r requirements.txt

# Or install as editable package
pip install -e .

# Run directly
python3 vapt_tool.py --help
python3 vapt_tool.py server   # starts on 0.0.0.0:8000
```

### 3.3 Uninstall

```bash
# Via installer
./install.sh --uninstall

# Manual
sudo systemctl stop vapt-tool vapt-api vapt-worker vapt-beat
sudo rm -rf /opt/vapt-tool
sudo rm /usr/local/bin/vapt
```

## 9. Deployment

### 9.1 Production Checklist

- [ ] Run `sudo ./install.sh` on target host
- [ ] Configure SSL: `sudo certbot --nginx`
- [ ] Change `config.yaml` credentials — rotate `secret_key`, `jwt_secret`, DB passwords
- [ ] Set `debug: false` and `environment: production`
- [ ] Configure firewall: only ports 22, 80, 443 open
- [ ] Enable 2FA / authentication for web UI

## 10. Troubleshooting

### Port already in use

```bash
# Find what's using the port
sudo lsof -i :8000
sudo ss -tlnp | grep 8000

# Change port in config.yaml or env var
VAPT_PORT=8080 ./vapt_tool.py server
```

### Permission denied on `./vapt_tool.py`

```bash
chmod +x vapt_tool.py
```

### Config file not found (expected in dev)

```text
WARNING - Config file not found: /opt/vapt-tool/config/config.yaml — using defaults
```

This is normal when running `vapt_tool.py` from the project directory without the full install. The tool falls back to safe defaults.

### Celery / Redis not available

```text
WARNING - Celery not installed — running without async task queue
```

Install the optional dependency group:
```bash
pip install -e ".[workers]"
redis-server  # start Redis locally
```

### Scanner returns no findings

All scanners are currently **stub implementations**. They return empty result sets until actual scanning logic is implemented (see §11).

## 11. Extending the Tool

### Adding a New Scanner

1. Create class in `vapt_tool.py` (or add to `src/scanners/<type>/scanner.py`):

```python
class DNSScanner:
    def run(self, target: str) -> dict[str, Any]:
        logger.info("Running DNS scan on %s", target)
        # ... implement scanning logic ...
        return {
            "target": target,
            "type": "dns",
            "status": "completed",
            "records": [],
            "summary": {"critical": 0, "high": 0, "medium": 0},
        }
```

2. Add to the scanner map in `run_scan()`:
```python
_SCANNER_MAP: Dict[str, Any] = {
    "web": WebScanner,
    "network": NetworkScanner,
    "api": APIScanner,
    "cloud": CloudScanner,
    "dns": DNSScanner,  # NEW
}
```

3. Expose in CLI (automatic if registered in `_SCANNER_MAP` — the `--type` choices list updates):
```bash
./vapt_tool.py scan example.com --type dns
```

### Adding a New API Endpoint

```python
def create_app() -> FastAPI:
    app = FastAPI(...)
    app.add_middleware(...)

    @app.get("/")
    async def _root():
        return {"status": "running"}

    # NEW: Add scan endpoint
    @app.post("/scan")
    async def _scan(request: ScanRequest):
        from vapt_tool import WebScanner
        result = WebScanner().run(request.target)
        return result

    return app
```

### Adding a New Dependency

1. Add to `pyproject.toml` under `[project.optional-dependencies]`
2. Add to `requirements.txt` in the appropriate section
3. Use lazy import pattern in code:
```python
try:
    import new_library
except ImportError:
    logger.warning("new_library not installed — feature disabled")
```
